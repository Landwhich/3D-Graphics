Assignment 02 - Too Technical

Summary of Task

The goal of assignment 2 was to create a Maya GUI scripting tool in python, to generate Lego Technic pieces. These pieces needed to reflect Lego's specific sizing so that they fit together correctly when manually assembled.

Method / Approach

My approach was based on reusing a set of global unit variables at the top of the script, derived from real Lego Technic measurements. All functions reference these globals, ensuring that every piece is consistent and physically compatible:

  • g_brickWidthUnit = 0.8 — 1x1 brick spacing
  • g_brickHeightUnit = 0.32 — thin plate height
  • g_studRadUnit = 0.25 — stud radius
  • g_holeRadUnit = 0.26 — technic/pin hole radius
  • g_axleUnit = 0.10 — axle arm thickness
  • g_axleHoleUnit = 0.12 — axle hole slot width

Each function follows the same general pattern: geometry is created and named within a namespace, a Lambert shader is created and assigned, and then the namespace is removed and merged back into the root. This pattern keeps the scene outliner clean regardless of how many pieces are generated. I tried to balance code reuse with optimal function structure. Some functions are parameterized if they can be, otherwize a single-use button was implemented for unique pieces

The required pieces implemented were:

  1. Standard Brick — a polyCube body with a grid of polyCylinder studs, united with polyUnite. Width, depth, and height are all driven by the GUI sliders.
  2. Sloped Brick — same base as the standard brick but with vertex displacement applied post-creation to collapse the top face into a slope. Supports 2, 3, and 4 stud depths with conditional vertex logic for each case.
  3. Holed Technic Liftarm — a bevelled polyCube with polyCylinder cutters subtracted via polyCBoolOp. Length is driven by an option menu in the GUI.
  4. Angled Liftarm — two bevelled arm bodies united together, with arm B rotated around a shared junction point. Hole cutters are placed along each arm's direction using trigonometry (sin/cos of the bend angle) and subtracted in a single boolean pass.
  5. Technic Axle — two thin polyCube cross-section bars rotated 90 degrees relative to each other and united, forming the characteristic plus-sign cross section of a Lego Technic axle.
  6. Technic Pin — a hollow cylinder created by subtracting a smaller inner cylinder from a larger outer one using polyCBoolOp.
  7. Technic Tire + Rim — the tire is built as an outer cylinder minus an inner hollow, with 16 tread knob cubes placed radially and united. The rim is a separate cylinder with a cross-shaped axle hole subtracted, sized to g_axleHoleUnit.
  8. Curved L-Liftarm — a square base with a cylinder subtracted to create the curved corner profile, then further booleans are used to add axle holes at the correct Technic spacing.
  9. 2x1 Axle Connector — a small rectangular body with two cross-shaped axle holes subtracted, one at each end, using unioned pairs of thin cubes as cutters.

One notable challenge was the polyCBoolOp function, which was unreliable when object transforms had not been frozen beforehand. The consistent fix was to call cmds.makeIdentity(apply=True, t=True, r=True, s=True) on all objects immediately after any transform, before passing them to the bool cut. This fixed the majority of boolean failures encountered during development.

Additional Information

Code optimization was a conscious focus throughout. Rather than duplicating geometry creation logic, shared patterns (bevel, move, freeze, boolean) were kept the same for all functions. The makeAngledLiftarm function in reuses the same hole-cutting pattern as makeHoledTechnic but extends it with trig offset calculations to place the bool cutters along the rotated arm B direction.

Boolean operations were kept to single multi-cutter calls where possible (passing all hole cylinders at once to one polyCBoolOp call) rather than chaining multiple sequential booleans, which both improves performance and reduces the chance of geometry errors accumulating. The same goes for multiple instances of beveling.
